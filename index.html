Démarreur CSS

Le démarreur CSS contient des composants et des modèles pour créer une interface utilisateur pure CSS. Ce n'est pas un cadre (framework) CSS !

Pour créer un site avec le démarreur CSS, commencez à zéro avec seulement un reset et un fichier CSS vierge. Construisez vos styles au fur et à mesure que vous construisez votre balisage. Tirez les styles, le balisage et les motifs du démarreur selon vos besoins. Comme vos styles diffèrent des styles de démarrage, modifiez les styles plutôt que de les surmoder.

Voici d'excellentes ressources pour créer et maintenir des bases de code CSS propres.

Thème

Le fichier theme.css contient des couleurs, des icônes et d'autres variables qui forment le thème de ce site.

Variables

Les variables de thème sont définies sur :root afin qu'elles puissent être utilisées dans tous les éléments. Les variables de couleur sont en haut des styles :root. Ceux-ci définissent toutes les couleurs à utiliser dans les fichiers CSS. Les variables de couleur ne doivent être utilisées que dans le fichier de thème et ne doivent pas être importées d'autres fichiers CSS. Fichiers.

Sous la variable de couleur, nous définissons des variables comme --text-color et --border-color que nous avons l'intention d'utiliser dans d'autres fichiers CSS. Ces variables doivent faire référence aux variables de couleur et définir notre thème par défaut. Nous pouvons définir un thème alternatif (avec @media (prefers-color-scheme : dark), peut-être) en remplaçant ces variables. 

:root {
    /* Color variables */
    --white: #EEEEEE;
    --black: #333333;

    /* Variables de thème par défaut */
    --text-color: var(--black);
    --background-color: var(--white);
}

@media (prefers-color-scheme: dark) {
    :root {
        /* Dark theme variables */
        --text-color: var(--white);
        --background-color: var(--black);
    }
}

Thème du navigateur

Définissez la propriété accent-color sur la balise html afin que les cases à cocher et les contours correspondent au thème de votre site. 

html {
    accent-color: var(--primary);
}

Ajoutez une balise méta qui définit la couleur du thème sur la tête de votre document pour styliser le navigateur chrome lorsqu'un utilisateur visite votre site. 

<meta name="theme-color" content="#196EC2"/>

Icons

Utilisez les icônes SVG chaque fois que c'est possible et pratique. Ils peuvent évoluer à n'importe quelle taille, accepter les styles CSS et sont généralement plus petits que les autres formats d'image.

Variables

Pour utiliser les variables CSS de notre thème dans les icônes SVG, nous devons inclure les SVG dans la page un peu différemment. La méthode habituelle sur une balise d'image ne fonctionne pas, car les SVG ne font pas partie du DOM. 

<img src="/images/spinner.svg" alt="loading">

Au lieu de cela, nous devons inclure le SVG complet directement avec une balise ou une référence <svg> à partir d'u <svg> tag utilisant une balise <use>. 

<svg>
    <use xlink:href="/images/spinner.svg#icon"></use>
</svg>

Cela garantit que le SVG fait partie du DOM, afin de pouvoir utiliser nos variables CSS. Si nous utilisons les variables CSS de notre thème pour styliser nos SVG, nous pourrons utiliser les mêmes icônes pour différents thèmes (mode clair et sombre, par exemple).

Sprites

Ce site utilise un fichier sprite SVG pour inclure de nombreux SVG dans un seul fichier. Cela aide un peu avec les performances puisque le navigateur fait moins de demandes de chargement d'icônes et peut aider lors de la modification de plusieurs icônes à la fois. Il est très lourd de référencer les sprites SVG dans un fichier CSS, nous utilisons donc des fichiers séparés pour les SVG référencés à partir de CSS.

Favicon

Le démarreur utilise un favicon SVG pour les navigateurs modernes et fournit un repli ico pour les navigateurs sans prise en charge des favicon SVG (comme Safari et IE). Le SVG favicon utilise une requête @media (prefers-color-scheme : dark) pour servir un favicon de couleur différente en fonction du schéma de couleurs de l'utilisateur. Le ico favicon se compose d'une icône sombre sur un fond clair afin qu'elle s'affiche bien sur n'importe quelle couleur d'arrière-plan.

Barre latérale

La barre latérale du démarreur suit un modèle vu dans de nombreuses applications. Sur les écrans larges, la barre latérale est affichée en permanence sur le côté gauche de la page et le contenu de la page est affiché à sa droite. Sur les écrans plus étroits, la barre latérale est masquée par défaut et ne s'affiche que lorsqu'un utilisateur clique sur l'icône du menu.

Cette mise en page est obtenue avec CSS Grid, en particulier zones de modèles de grille. La mise en page par défaut définit les zones pour chacun des éléments de mise en page de niveau supérieur, en plaçant le nav dans la zone sidebar. 

body {
    ...
    grid-template-areas: 'header header'
                         'sidebar main'
                         'footer footer';
}

nav {
    grid-area: sidebar;
}

Sur les petits écrans, nous omettons la barre latérale de la disposition de la grille et positionnons la barre latérale de manière absolue, en utilisant la propriété left pour basculer sa visibilité. 

@media screen and (max-width: 800px) {
    body {
        ...
        grid-template-areas: 'header'
                             'main'
                             'footer';
    }

    nav {
        grid-area: unset;
        position: absolute;
        top: var(--header-height);
        bottom: 0;
        left: calc(-1 * var(--nav-width));
    }

    .menu-open nav {
        left: 0;
    }
}
